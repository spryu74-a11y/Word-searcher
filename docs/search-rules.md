# 끝말잇기 검색 규칙

## 입력 정규화

- 검색어와 단어장 항목은 유니코드 NFC 기준으로 정규화한다.
- 검색어의 앞뒤 공백과 중간 공백은 제거한다.
- 한글 완성형 음절(`가`-`힣`)은 그대로 읽기로 사용한다.
- 영문 알파벳만 있는 검색어는 내장 발음 변환표로 한글 읽기를 만든다.
- 숫자, 특수문자, 한글 자모만 있어 읽기를 만들 수 없는 입력은 검색하지 않고 입력 오류로 처리한다.
- 검색어 최대 길이는 80자다.

## 끝말잇기 연결

- `starts` 모드는 입력 읽기로 시작하는 단어를 찾는다.
- `reply` 모드는 입력 읽기의 마지막 글자를 끝 글자로 보고, 그 글자로 시작할 수 있는 응답 단어를 찾는다.
- 마지막 글자는 정규화된 읽기 문자열의 마지막 한글 음절이다.
- 두음법칙은 시작 가능한 글자 목록으로 처리한다.
  - `ㄹ` 시작 음절은 모음 조건에 따라 `ㄴ` 또는 `ㅇ` 시작 변형을 허용한다.
  - `ㄴ` 시작 음절 중 일부 이중모음 계열은 `ㅇ` 시작 변형을 허용한다.
- 예: `계란`의 응답 시작 글자는 `란`, `난`이다.

## 한방/대체/블런더 판정

- 입력/출력 계약:
- 입력: 정규화된 단어 항목, 읽기, 시작 글자, 끝 글자, 사용된 단어 목록.
  - 출력: `oneShot`, `alternativeOneShot`, `blunder`, `followerCount`, `oneShotReplyCount`, `alternativeOneShotReplyCount`.
- 한방단어는 상대가 이어갈 수 있는 후속 단어 수가 0인 단어다.
- 대체 한방단어는 직접 한방은 아니지만, 모든 유효 후속 단어가 다시 막히는 계열일 때 표시한다.
- 블런더는 이어갈 단어가 있으나 상대에게 한방 또는 대체 한방 반격을 허용하는 단어다.
- 방금 낸 단어를 다시 쓸 수 없게 되어, 상대가 원래 시작 글자로 되돌린 뒤 남은 후속어가 모두 블런더가 되는 경우도 블런더로 분류한다.
- 사용됨으로 표시한 단어는 후속어 수에서 제외한다. 따라서 남은 후속어가 0개가 되면 해당 단어는 한방으로 바뀐다.

## 오류와 빈 결과

- 결과가 없으면 오류로 보지 않고 "조건에 맞는 단어가 없습니다"를 표시한다.
- 검색 엔진, 네트워크, 인덱스 로딩 실패는 사용자용 한국어 메시지와 재시도 버튼을 표시한다.
- 예상과 다른 응답 구조는 안전한 기본값으로 보정하고, 보정 내역은 개발자 로그에 경고로 남긴다.

## 사용됨 표시

- `사용됨` 상태는 `usedWordIds`에 저장하고, 후속어 수와 한방 판정에 반영한다.
- 원본 단어장과 검색 인덱스는 바꾸지 않으며, 사용됨 단어 자체는 결과에 흐리게 계속 표시한다.
- 후속어 수가 바뀐 결과는 페이지 단위로 다시 정렬하고, 렌더링 시 `isUsed`를 합성해 체크박스와 흐림 스타일을 표시한다.
- 사용됨 식별자는 배열 index가 아니라 NFC, 공백 제거, 소문자 정규화를 거친 단어 id를 사용한다.
